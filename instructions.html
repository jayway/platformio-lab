PlatformIO

an open source ecosystem for IoT development

Introduction

Some of the nice features of PlatformIO:

  • Cross-platform build system (works on Windows, Linux and OSX)
  • Contains a library manager that automatically pulls down your dependencies
  • Has a powerful CLI, see http://platformio.org/get-started/cli
  • Has an nice IDE based on Atom, an open source and easily extendable text editor, see http://platformio.org/get-started/ide
Plus many more. Have a look at it here: PlatformIO

Preperations

  • Install PlatformIO
  • Install Wemos D1 mini drivers if needed (only Mac & Windows). Note that older versions of the driver will cause a kernel panic on Mac. Drivers can be found here and here.

Part 1 - trying it out

We have 2 different boards to play around with; the classic Arduino as well as the Wemos D1 mini, a mini WiFi board based on ESP8266.

Lab - Arduino UNO

Purpose: The Arduino should be able to communicate with the "world" using one of the shields, e.g. Network, NFC, etc. There should of course be some leds blinking as well.

There is example code available in the repository. It is divided in classes, which in some cases doesn't make sense, but it allows for easy categorization and running.

Lab - Wemos D1 mini

Purpose: The Wemos should be able to connect to the local Wifi. And there should be some leds blinking as well...

There is example code available in the repository. It is divided in classes, which in some cases doesn't make sense, but it allows for easy categorization and running.
